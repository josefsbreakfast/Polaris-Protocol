Leaks don‚Äôt just *happen*. They trace the outlines of the systems they escape from.

## ‚ú® The Purpose of LVSICR  

**LVSICR** stands for:  
- **L ‚Äì Leak (Primary Source)**  
- **V ‚Äì Vector (Transmission Path)**  
- **S ‚Äì Spread (Systemic Reach)**  
- **I ‚Äì Intent (Motivation or Directionality)**  
- **C ‚Äì Containment (System Response)**  
- **R ‚Äì Recurrence (Temporal Memory)**  

Each axis has four stages:  
`0 = dormant`, `1 = mild/local`, `2 = moderate/patterned`, `3 = critical/systemic`.  
Together they form a **12-digit behavioural signature**: six letters, six numbers.  

> ‚ÄúL2 V1 S1 I2 C1 R2‚Äù conveys in two seconds what a full incident paragraph might take a page to explain.

## üéº LVSICR Progressive Staging Guide  

*(From local anomaly to systemic recurrence)*  

### **L ‚Äî Leak (Primary Source)**  
| Stage | Descriptor | Observable Behaviour | Example Scenario |
|:--|:--|:--|:--|
| **L0 ‚Äì Latent** | Potential leak exists but not yet active. | Diagnostic noise, incomplete log. | Debug file with personal data still internal. |
| **L1 ‚Äì Localised** | Discrete fault. | Single email mis-sent. | Spreadsheet shared to wrong list. |
| **L2 ‚Äì Patterned / Recurrent** | Design flaw emerging. | Same field leaks every month. | Recurring report exposure. |
| **L3 ‚Äì Structural / Critical** | Built-in architectural failure. | Cannot fix without redesign. | Platform logs user data to public cache. |

### **V ‚Äî Vector (Transmission Path)**  
| Stage | Descriptor | Observable Behaviour | Example Scenario |
|:--|:--|:--|:--|
| **V0 ‚Äì Contained** | No outward travel. | Quarantined immediately. | QA team deletes bad file. |
| **V1 ‚Äì Internal Spread** | Moves across teams or systems. | Shared via email or mirror. | HR file copied to Finance. |
| **V2 ‚Äì Cross-Institutional** | Crosses organisational boundary. | Supplier or API bleed. | Contractor downloads client records. |
| **V3 ‚Äì Public / Viral** | Fully external. | Public upload or press release. | Document on social media. |

### **S ‚Äî Spread (Systemic Reach)**  
| Stage | Descriptor | Observable Behaviour | Example Scenario |
|:--|:--|:--|:--|
| **S0 ‚Äì None** | Self-contained. | Single dataset only. | Local log error. |
| **S1 ‚Äì Local Cluster** | Limited departmental spread. | Cluster of linked records. | Council division leak. |
| **S2 ‚Äì Institutional / Sectoral** | Multiple sites or agencies. | Cascade of breaches. | University-wide fault. |
| **S3 ‚Äì Ecosystemic** | Cross-sector chain reaction. | Regulatory failure + trust loss. | Cloud vendor leak impacting clients globally. |

### **I ‚Äî Intent (Motivation / Directionality)**  
| Stage | Descriptor | Observable Behaviour | Example Scenario |
|:--|:--|:--|:--|
| **I0 ‚Äì Accidental** | Pure error. | Unaware user action. | Mis-click send-all. |
| **I1 ‚Äì Negligent** | Warnings ignored. | Deferred patch. | Known issue unfixed. |
| **I2 ‚Äì Structural** | Leak baked into process. | Policy gap or perverse incentive. | Open data without consent review. |
| **I3 ‚Äì Deliberate** | Leak used as weapon or activism. | Whistle or dox. | Insider exports file to harm or reveal. |

### **C ‚Äî Containment (System Response)**  
| Stage | Descriptor | Observable Behaviour | Example Scenario |
|:--|:--|:--|:--|
| **C0 ‚Äì Automatic Resolution** | Self-correcting system. | Auto-revoke link. | Expiring URL. |
| **C1 ‚Äì Passive Denial** | No fix; minimise story. | ‚ÄúNothing to see here.‚Äù | Press line only. |
| **C2 ‚Äì Technical Patch** | Fix without learning. | Silent config change. | Quick patch cycle. |
| **C3 ‚Äì Active Suppression** | Attack messenger. | Gag or discipline. | Whistleblower sacked. |

### **R ‚Äî Recurrence (Temporal Memory)**  
| Stage | Descriptor | Observable Behaviour | Example Scenario |
|:--|:--|:--|:--|
| **R0 ‚Äì One-off** | Singular event. | Random glitch. | No pattern. |
| **R1 ‚Äì Periodic** | Re-emerges cyclically. | Quarterly incident. | Regular dump window. |
| **R2 ‚Äì Chronic** | Persistent fault line. | Always leaking somewhere. | Legacy code exposes metadata. |
| **R3 ‚Äì Mutating / Adaptive** | Evolves with counter-measures. | Fixes spawn new forms. | Fork recreates vulnerability. |
