## ğŸ—‚ï¸ Terminology Note â€” Data Governance  

This node uses **data cooperative** deliberately.

Several adjacent terms circulate in civic tech spaces; they are *not* interchangeable:

- **Data Cooperative** â€” *preferred*  
  Member-owned and democratically governed data structures.  
  One-member-one-vote (or equivalent), collective benefit, explicit limits on reuse and extraction.

- **Data Trust**  
  A fiduciary stewardship model where trustees manage data on behalf of a defined group.  
  Strong for public-sector or regulatory contexts; less participatory day-to-day.

- **Data Commons**  
  A broader normative frame emphasising shared access and collective norms.  
  Useful conceptually; often underspecified legally.

- **DAO (Decentralised Autonomous Organisation)**  
  Originating in web3 contexts.  
  Can support shared governance, but frequently drifts toward financialisation and exclusion unless tightly constrained.

Where external projects use inconsistent language, this folder treats **data cooperatives** as the anchor term â€” with trusts or commons referenced only where structurally accurate.
