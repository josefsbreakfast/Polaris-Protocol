# 🗂️ Polaris Folder Routing Notes  
**First created:** 2026-05-19 | **Last updated:** 2026-05-19  

*Working routing logic for the evolving Polaris folder architecture. Intended as a practical placement guide and onboarding shorthand for future chats and repo organisation.*

---

## 🛰️ Core Principle  

The primary numbered folders are not just topic categories.  

They represent a rough progression through:

1. understanding systems,
2. recognising patterns,
3. orienting within live conditions,
4. applying practical interventions,
5. and developing long-term survivability or reconstruction strategies.

This creates a more intuitive navigation structure than flat taxonomies alone.

---

# 🌑 Folder 1 — Origin Points  
`🌑_1_Origin_Points`

## 🧭 Function  

Foundational frameworks, world models, structural analysis, and meta-layer conceptual systems.

This is the:
> “What is this project actually seeing?” layer.

Focus:
- systemic explanation,
- governance theory,
- containment logic,
- legitimacy structures,
- civilisational patterns,
- foundational diagnostics,
- conceptual architecture.

## 📂 Example Contents  

- 🧄 Exousiología
- 🌸 Containment Studies
- 🌀 System Governance
- Big Picture Protocols
- governance repair frameworks
- foundational README architecture
- meta-analysis nodes
- legitimacy diagnostics
- structural ecology concepts

## 🧠 General Rule  

If a node primarily explains:
- *why systems behave as they do*,
- *how structures emerge*,
- or *what conceptual lens Polaris is using*,

it probably belongs in Folder 1.

# 🌓 Folder 2 — Signs & Symptoms  
`🌓_2_Signs_Symptoms`

## 🧭 Function  

Pattern recognition, anomaly detection, weirdness screening, suppression signals, and early diagnostic frameworks.

This is the:
> “Something feels wrong but I cannot articulate it yet.” layer.

Focus:
- detection,
- symptom recognition,
- metadata anomalies,
- narrative inconsistencies,
- suppression signatures,
- containment indicators,
- weirdness classification.

## 📂 Example Contents  

- weirdness screening
- metadata sabotage networks
- containment scripts
- syntax bombs
- apparitional objects
- suppression pattern recognition
- narrative drift analysis
- visibility manipulation systems
- anomaly tracking

## 🧠 General Rule  

If a node primarily helps users:
- notice patterns,
- identify manipulation,
- recognise containment,
- or classify anomalies,

it likely belongs in Folder 2.

## ⚠️ Notes  

Some folders may partially drift elsewhere later.

Example:
- “data risks” may eventually split between:
  - operational safety,
  - infrastructure governance,
  - and metadata analysis.

Routing remains flexible.

# 🌗 Folder 3 — In The Moment  
`🌗_3_In_The_Moment`

## 🧭 Function  

Interpretive tools, situational navigation, behavioural decoding, and live-condition orientation systems.

This is the:
> “I am inside the weather system now. How do I orient myself?” layer.

Focus:
- interpretation,
- behavioural mapping,
- emotional navigation,
- social decoding,
- cognitive orientation,
- live-environment understanding.

## 📂 Example Contents  

- ⚗️ Apothecary
- temperament systems
- constitutions
- vapours
- behavioural decoding tools
- social pattern interpretation
- meaning-making frameworks
- adaptive orientation systems

## 🧠 General Rule  

If a node primarily helps users:
- understand live conditions,
- orient socially or psychologically,
- decode behaviour,
- or navigate active environments,

it likely belongs in Folder 3.

## ⚠️ Important Distinction  

Folder 3 is not primarily:
- emergency intervention,
- or long-term strategy.

It is:
> interpretation and orientation within active conditions.

# 🌖 Folder 4 — Small Steps  
`🌖_4_Small_Steps`

## 🧭 Function  

Practical adaptation, applied interventions, countermeasures, resilience-building, and actionable survival tools.

This is the:
> “What can I actually DO right now?” layer.

Focus:
- practical tools,
- workflow changes,
- countermeasures,
- adaptive systems,
- immediate stabilisation,
- operational resilience.

## 📂 Example Contents  

- ❤️‍🩹 Survivor Tools
- 🦆 Digital Disruption
- practical countermeasures
- workflow adaptations
- cognitive support systems
- environmental stabilisation
- repair practices
- resilience protocols

## 🧠 General Rule  

If a node primarily contains:
- practical actions,
- implementation methods,
- stabilisation strategies,
- or survivability tools,

it likely belongs in Folder 4.

## ⚠️ Important Distinction  

Folder 4 is intentionally separated from Folder 3.

Otherwise:
- interpretation,
- analysis,
- and intervention

collapse together and create recursive “analysis paralysis” structures.
